DOKUMENTASI CYBER SECURITY LOG ANALYZER
=====================================

1. PENDAHULUAN
-------------
Cyber Security Log Analyzer adalah aplikasi yang dirancang untuk menganalisis log aktivitas jaringan dengan tujuan mendeteksi aktivitas mencurigakan dan potensial serangan cyber. Aplikasi ini mengimplementasikan berbagai struktur data fundamental dalam konteks keamanan siber.

2. STRUKTUR DATA YANG DIGUNAKAN
------------------------------
2.1 Array (vector<LogEntry>)
    - Implementasi: vector<LogEntry> logArray
    - Fungsi: Menyimpan semua log aktivitas jaringan
    - Kelebihan: 
      * Akses cepat dan random untuk semua log
      * Memudahkan pencarian dan pengolahan data
      * Efisien untuk penyimpanan data berurutan

2.2 Queue (queue<LogEntry>)
    - Implementasi: queue<LogEntry> logQueue
    - Fungsi: Antrian pemrosesan log yang masuk (FIFO)
    - Kelebihan:
      * Menjamin log diproses secara berurutan
      * Mengikuti prinsip First In First Out
      * Ideal untuk pemrosesan data berurutan

2.3 Stack (stack<LogEntry>)
    - Implementasi: stack<LogEntry> alertStack
    - Fungsi: Melacak history aktivitas mencurigakan (LIFO)
    - Kelebihan:
      * Memudahkan akses ke alert terbaru
      * Mengikuti prinsip Last In First Out
      * Efisien untuk melacak aktivitas terakhir

2.4 Linked List (SuspiciousIPList)
    - Implementasi: Class SuspiciousIPList dengan nodes yang terhubung
    - Fungsi: Menyimpan daftar IP yang mencurigakan
    - Kelebihan:
      * Efisien untuk operasi penambahan dan penghapusan
      * Memori dinamis
      * Mudah untuk memperbarui data

2.5 Tree (AttackClassifier)
    - Implementasi: Class AttackClassifier dengan struktur pohon keputusan
    - Fungsi: Klasifikasi tingkat bahaya berdasarkan atribut log
    - Kelebihan:
      * Pengambilan keputusan terstruktur
      * Mudah untuk menambah aturan baru
      * Efisien dalam klasifikasi

3. KOMPONEN UTAMA
----------------
3.1 Struktur LogEntry
    - Menyimpan informasi log:
      * timestamp: Waktu kejadian
      * ipAddress: Alamat IP
      * action: Aksi yang dilakukan
      * status: Status aksi

3.2 Class SuspiciousIPList
    - Fungsi:
      * Menambah IP mencurigakan
      * Menampilkan daftar IP
      * Mencari IP
      * Menghapus IP
    - Implementasi menggunakan linked list

3.3 Class AttackClassifier
    - Fungsi:
      * Klasifikasi serangan
      * Pembuatan pohon keputusan
      * Evaluasi tingkat bahaya
    - Menggunakan decision tree

3.4 Class LogAnalyzer
    - Fungsi utama aplikasi:
      * Manajemen log
      * Pemrosesan log
      * Analisis keamanan
      * Penyimpanan dan pembacaan file

4. ALUR KERJA APLIKASI
---------------------
1. Log aktivitas jaringan diterima dan disimpan dalam array
2. Log dimasukkan ke dalam queue untuk diproses
3. Log diproses menggunakan decision tree untuk klasifikasi
4. Jika log mencurigakan:
   - IP ditambahkan ke linked list IP mencurigakan
   - Log ditambahkan ke stack alert
5. User dapat melihat IP mencurigakan dan alert terbaru

5. FITUR UTAMA
-------------
5.1 Manajemen Log
    - Penambahan log manual
    - Pembacaan log dari file
    - Penyimpanan log ke file
    - Tampilan semua log

5.2 Analisis Keamanan
    - Pemrosesan log dalam queue
    - Klasifikasi tingkat bahaya
    - Pelacakan IP mencurigakan
    - Tampilan alert terbaru

5.3 Visualisasi
    - Tampilan decision tree
    - Daftar IP mencurigakan
    - Alert terbaru
    - Semua log

6. PANDUAN PENGGUNAAN
--------------------
6.1 Menambahkan Log
    - Manual: Pilih menu 1
    - Dari File: Pilih menu 2
    Format file: CSV dengan kolom:
    timestamp,ipAddress,action,status

6.2 Memproses Log
    - Pilih menu 3
    - Log akan diproses secara berurutan
    - Hasil klasifikasi ditampilkan

6.3 Melihat Informasi
    - Alert Terbaru: Menu 4
    - Semua Log: Menu 5
    - IP Mencurigakan: Menu 6
    - Decision Tree: Menu 7

6.4 Menyimpan Log
    - Pilih menu 8
    - Masukkan nama file
    - Log akan disimpan dalam format CSV
